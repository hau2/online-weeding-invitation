  <action>
    Create `PublishButton.tsx` -- 'use client' component handling the full publish/unpublish lifecycle.

    **Props:**
    ```typescript
    interface PublishButtonProps {
      invitation: Invitation
      onPublished: (updated: Invitation) => void   // callback to update parent state
      onUnpublished: (updated: Invitation) => void  // callback to update parent state
      onPreview?: () => void                         // callback to open full preview dialog (wired by EditorShell)
    }
    ```

    **State management:**
    - `showConfirmDialog: boolean` -- first-publish confirmation
    - `showCelebrationDialog: boolean` -- post-publish celebration
    - `publishing: boolean` -- loading state
    - `unpublishing: boolean` -- loading state

    **Publish flow (per user decision):**
    1. User clicks "Xuat ban" button (prominent rose/pink in topbar)
    2. Confirmation dialog opens:
       - If invitation has NO slug (first publish): show message warning that the link is permanent ("Duong link se khong the thay doi sau khi xuat ban"), show the slug that will be generated (we don't know it yet, so say "He thong se tao duong link tu dong")
       - If invitation HAS slug (re-publish after unpublish): show message "Ban muon xuat ban lai thiep cuoi?" with existing URL
       - Confirm button: "Xuat ban" (rose-500 bg)
       - Cancel button: "Huy"
    3. On confirm: call `apiFetch('/invitations/${invitation.id}/publish', { method: 'POST', credentials: 'include' })`
    4. On success: close confirm dialog, call `onPublished(data)` to update parent state
    5. If this was first publish (invitation had no slug before): open celebration dialog
    6. If re-publish: just close dialog (no celebration)

    **Celebration dialog (first publish only, per user decision):**
    - Show celebration message: "Thiep cuoi da duoc xuat ban!"
    - Fire canvas-confetti (dynamically imported via `next/dynamic` or `import()` to avoid SSR): realistic confetti burst from center
    - Display the public URL: `${window.location.origin}/w/${updatedInvitation.slug}`
    - Copy URL button: copies to clipboard, shows toast "Da sao chep lien ket"
    - "Chia se ngay" action button (for now, just copies link -- Phase 5 will add actual share)
    - "Dong" close button

    **canvas-confetti install:** The executor should run `cd apps/web && npm install canvas-confetti && npm install -D @types/canvas-confetti` before building this component. Import it dynamically: `const confetti = (await import('canvas-confetti')).default`

    **Post-publish button state:**
    - When `invitation.status === 'published'`: show green/teal "Da xuat ban" button instead of rose "Xuat ban"
    - Next to the button: a DropdownMenu trigger (three dots / MoreVertical icon)

    **Unpublish flow (per user decision -- hidden in more-menu):**
    1. DropdownMenu with "Huy xuat ban" option (only visible when status is 'published')
    2. Also add "Xem truoc" option in the dropdown -- when clicked, call `onPreview?.()` to notify EditorShell to open the full preview dialog
    3. On click "Huy xuat ban": show simple confirmation dialog ("Ban muon huy xuat ban thiep cuoi?")
    4. On confirm: call `apiFetch('/invitations/${invitation.id}/unpublish', { method: 'POST', credentials: 'include' })`
    5. On success: call `onUnpublished(data)` to update parent state, close dialog
    6. On error: toast.error with message

    **Validation before publish:**
    - Check if required fields are filled: brideName, groomName (minimum)
    - If empty: toast.error("Vui long nhap ten co dau va chu re truoc khi xuat ban") and don't open dialog
    - Other fields (date, venue, message) are optional -- couples can publish drafts

    IMPORTANT: The slug URL displayed in dialogs must use the UPDATED invitation data returned from the API (which contains the generated slug), not the pre-publish state.

    IMPORTANT: canvas-confetti must be dynamically imported (not top-level import) to avoid SSR issues and keep initial bundle small.
