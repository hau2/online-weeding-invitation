**Create `PhotoGallery.tsx`** as a 'use client' component.

Props interface:
```typescript
interface PhotoGalleryProps {
  invitationId: string
  photoUrls: string[]
  onChange: (photoUrls: string[]) => void
}
```

**Upload behavior:**
- Hidden file input (accept="image/jpeg,image/png,image/webp") with multiple=true, ref-triggered by "+" button
- The entire component acts as a drop target: use onDragOver/onDrop native events (NOT dnd-kit, those are for reorder) to detect file drops
- On drop/select: create FormData with field name "photos", append selected files
- Call `apiUpload<Invitation>(`/invitations/${invitationId}/photos`, formData)`
- On success: call `onChange(data.photoUrls)` to update parent state
- On error: show toast.error with the Vietnamese error message
- While uploading: show uploading state (overlay on new thumbnails or a spinner in the drop zone)
- Max 10 photos total: if photoUrls.length >= 10, disable the add button and show "Da dat toi da 10 anh"

**Drop zone overlay:**
- Track dragOver state with useState
- When files are dragged over the component, show a semi-transparent overlay with text "Tha anh vao day" and a dashed border
- Reset on dragLeave/drop

**Thumbnail grid with @dnd-kit sortable:**
- Use DndContext with closestCenter collision detection
- SortableContext with rectSortingStrategy (for 2D grid)
- Grid layout: `grid grid-cols-3 gap-2` (3 columns of square thumbnails)
- Each photo is a SortablePhoto sub-component using useSortable hook
- SortablePhoto renders: img (object-cover, aspect-square, rounded-lg), GripVertical drag handle icon (top-left, spread ...listeners and ...attributes on it), X delete button (top-right)
- CSS transform applied via CSS.Transform.toString(transform)

**Reorder behavior (CRITICAL -- uses dedicated endpoint, NOT auto-save):**
- onDragEnd handler: if active.id !== over.id, use arrayMove from @dnd-kit/sortable to reorder
- Call `onChange(newOrder)` to update local state immediately for responsive UI
- Then call `apiFetch<Invitation>(`/invitations/${invitationId}/photo-order`, { method: 'PATCH', body: { photoUrls: newOrder } })` to persist the reordered URLs
- This uses the dedicated PATCH :id/photo-order endpoint, NOT the generic PATCH auto-save, because photoUrls is intentionally excluded from FIELD_MAP
- On error: toast.error and revert to previous order

**Delete behavior:**
- X button calls `apiFetch<Invitation>(`/invitations/${invitationId}/photos/${index}`, { method: 'DELETE' })`
- On success: call `onChange(data.photoUrls)` to update parent
- On error: toast.error

**Photo IDs for dnd-kit:**
- dnd-kit needs string IDs. Use the URL itself as the ID (each URL is unique).
- items prop on SortableContext: photoUrls array directly

**Visual details (Claude's discretion):**
- Thumbnail size: aspect-square with rounded-lg corners
- Drag handle: GripVertical icon from lucide-react, size-4, semi-transparent white bg for contrast
- Delete button: X icon from lucide-react, size-4, red-500 bg circle
- Upload progress: simple opacity-50 overlay with a Loader2 spinning icon on each uploading thumbnail
- "+" add button: dashed border square in the grid, matching thumbnail size, with Plus icon

**Create `MusicPicker.tsx`** as a 'use client' component.

Props:
```typescript
interface MusicPickerProps {
  selectedTrackId: string | null
  onSelect: (trackId: string | null) => void
}
```

Behavior:
- On mount, fetch tracks: `apiFetch<SystemMusicTrack[]>('/invitations/music-tracks')` with credentials:'include'
- Render a list of tracks, each showing: title, artist (if present), duration formatted as M:SS
- Play/Pause button (Play or Pause icon from lucide-react) next to each track
- Clicking play: create Howl instance with src=[track.url] and html5:true, play it, set a 30-second timeout to auto-stop
- Clicking play on a different track: stop current, start new one
- Clicking pause: stop current playback
- Track currently playing gets a visual indicator (animated equalizer bars or pulsing icon)
- Selected track has a highlighted background (rose-50 or rose-100) and checkmark
- Clicking a track toggles selection: if already selected, deselect (onSelect(null)); if not selected, select (onSelect(track.id))
- Cleanup on unmount: unload Howl instance, clear timeout (prevent memory leak and continued playback)
- Use useRef for Howl instance and timeout -- do NOT store Howl in state
- Import Howl from 'howler' (default import)
