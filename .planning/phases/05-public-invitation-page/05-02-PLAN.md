<objective>
Set up ISR revalidation infrastructure and OG meta tag compatibility for Zalo/Facebook crawlers.

Purpose: When a couple publishes or unpublishes their invitation, the cached public page must be invalidated. Zalo and Facebook crawlers need OG tags in the initial HTML response (not streamed), so we configure htmlLimitedBots. These are infrastructure pieces that the public page (Plan 05-03) depends on.
Output: Revalidation route handler, NestJS revalidation trigger, next.config.ts with Zalo bot support.
</objective>

  <action>
    1. Create apps/web/app/api/revalidate/route.ts:
       - Export async POST handler
       - Read `x-revalidation-secret` header from request
       - Compare against process.env.REVALIDATION_SECRET
       - If mismatch, return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })
       - Parse request body for { slug }
       - Call revalidateTag(`invitation-${slug}`)
       - Return NextResponse.json({ revalidated: true })
       - Import revalidateTag from 'next/cache', NextRequest/NextResponse from 'next/server'

    2. Add triggerRevalidation(slug: string) private method to InvitationsService:
       - Get NEXT_PUBLIC_URL from ConfigService (fallback: 'http://localhost:3000')
       - Get REVALIDATION_SECRET from ConfigService
       - If no REVALIDATION_SECRET configured, skip silently (dev mode)
       - POST to `${nextUrl}/api/revalidate` with JSON body { slug } and x-revalidation-secret header
       - Wrap in try/catch -- revalidation failure is non-blocking (log warning, don't throw)
       - Use global fetch (available in Node 18+)

    3. Wire triggerRevalidation into existing publish() and unpublish() methods:
       - After successful publish (both first publish and re-publish): await this.triggerRevalidation(slug)
       - After successful unpublish: await this.triggerRevalidation(invitation.slug) (slug is preserved on unpublish)
       - Place after the DB update but before returning the response
  </action>
