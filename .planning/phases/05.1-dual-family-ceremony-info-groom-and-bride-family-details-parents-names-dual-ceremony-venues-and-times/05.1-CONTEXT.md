# Phase 5.1: Dual-Family Ceremony Info + Love Story - Context

**Gathered:** 2026-03-15
**Status:** Ready for planning

<domain>
## Phase Boundary

Add dual-family ceremony information (groom's and bride's family details, parents' names, separate venues/dates/times) and a love story timeline to the invitation. Includes database schema changes, editor form updates, template rendering, public page display with side-specific URLs (`?side=groom` / `?side=bride`), and dashboard sharing UI updates. Replaces the existing single ceremony fields with per-family ceremony fields.

</domain>

<decisions>
## Implementation Decisions

### Family Info Fields
- Each family side (Nhà trai / Nhà gái) has:
  - Father name (Cha)
  - Mother name (Mẹ)
  - Ceremony venue name
  - Ceremony venue address
  - Ceremony date (fully independent per side — can be different days)
  - Ceremony time (fully independent per side)
- The existing single weddingDate/weddingTime/venueName/venueAddress fields are REPLACED by per-family fields — not kept alongside

### Dual URL System
- Same slug, query parameter for side: `/w/{slug}?side=groom` and `/w/{slug}?side=bride`
- Default (no `?side=` param) shows groom's side
- Each side URL shows ONLY that side's info:
  - ✓ That side's parents' names
  - ✓ That side's ceremony venue/date/time
  - ✓ That side's bank QR code
  - ✓ Shared content: couple names, photos, love story, music, countdown
  - ✗ Other side's parents, venue, and bank QR are completely hidden
- Countdown timer uses that side's ceremony date (not a shared date)

### Love Story Timeline
- Structured timeline milestones (not free-text)
- Each milestone has: date (year or full date), title, description (short text)
- No photos in milestones — text only. Photos stay in the gallery section
- Up to 5 milestones max
- Timeline rendered as vertical line with dots on the public page
- Timeline styling matches template colors (gold on Traditional, rose gold on Modern, gray on Minimalist)
- Position on public page: after couple names, before ceremony details

### Editor Organization
- 9 accordion sections total (replacing old "Lễ cưới" with Nhà trai + Nhà gái, adding story):
  1. **Cặp đôi** (1/9): groom name, bride name
  2. **Nhà trai** (2/9): groom father, groom mother, ceremony date, time, venue name, venue address
  3. **Nhà gái** (3/9): bride father, bride mother, ceremony date, time, venue name, venue address
  4. **Câu chuyện tình yêu** (4/9): up to 5 milestones (date + title + description), add/remove buttons
  5. **Lời mời** (5/9): invitation message, thank-you text
  6. **Ảnh cưới** (6/9): photo gallery (existing)
  7. **Nhạc nền** (7/9): music picker (existing)
  8. **QR Ngân hàng** (8/9): bank QR upload (existing — already has groom + bride QR)
  9. **Bản đồ** (9/9): venue map URL (existing)

### Dashboard Sharing UI
- Two copy buttons on each published invitation card: "Link nhà trai" and "Link nhà gái"
- Each copies the respective URL with the `?side=` parameter
- Post-publish celebration dialog also shows both URLs

### Database Migration
- Remove old single ceremony fields (weddingDate, weddingTime, venueName, venueAddress) — migrate data to groom side fields
- Add per-family fields: groom_father, groom_mother, groom_ceremony_date, groom_ceremony_time, groom_venue_name, groom_venue_address, bride_father, bride_mother, bride_ceremony_date, bride_ceremony_time, bride_venue_name, bride_venue_address
- Add love_story JSONB column (array of milestone objects: `[{date, title, description}]`)

### Claude's Discretion
- Exact timeline dot/line CSS styling per template
- Milestone add/remove animation in editor
- How to handle migration of existing invitations (old fields → groom side)
- Error handling for empty family sections
- How the "Bản đồ" section works with two venues (per-side map URL or shared)

</decisions>

<specifics>
## Specific Ideas

- Vietnamese wedding invitations traditionally show both families — this is the culturally correct format
- The two-URL approach lets couples share the right link with the right guests (groom's family gets groom link, bride's family gets bride link)
- Love story timeline adds a personal touch — most couples have 3-4 milestones (met, dated, engaged, wedding)
- Bank QR already supports dual (groom + bride) from Phase 4 — this aligns with the per-side URL filtering

</specifics>
