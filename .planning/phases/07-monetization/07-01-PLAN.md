**4. Update create()** to explicitly set `plan: 'free', payment_status: 'none'` in the insert.

**5. Add free-tier publish guard** in both `publish()` and `publishSaveTheDate()`:
Before allowing publish, check if the invitation's plan is 'free'. If free, count how many other invitations this user has with status IN ('published', 'save_the_date') AND deleted_at IS NULL AND id != current invitation id. If count >= 1, throw BadRequestException with Vietnamese message: 'Ban chi co the xuat ban 1 thiep mien phi. Vui long nang cap len Premium de xuat ban them.'

**6. Add requestUpgrade() method:**
- Takes userId and invitationId
- Calls findOne to verify ownership
- If invitation.plan is already 'premium', throw BadRequestException: 'Thiep nay da la Premium'
- If invitation.paymentStatus is 'pending', throw BadRequestException: 'Yeu cau nang cap dang cho xu ly'
- Update payment_status to 'pending' in DB
- Return updated invitation

**7. Add adminApproveUpgrade() method:**
- Takes invitationId only (admin action, no userId scope)
- Fetch invitation directly by ID (no user scope - use supabaseAdmin.client directly)
- If not found, throw NotFoundException
- If payment_status is not 'pending', throw BadRequestException: 'Khong co yeu cau nang cap cho xu ly'
- Update: `{ plan: 'premium', payment_status: 'none' }`
- If slug exists, call triggerRevalidation(slug)
- Return updated invitation

**8. Add adminRejectUpgrade() method:**
- Takes invitationId only
- Fetch invitation directly by ID (no user scope)
- If not found, throw NotFoundException
- If payment_status is not 'pending', throw BadRequestException
- Update: `{ payment_status: 'rejected' }`
- Return updated invitation

**9. Add adminListPendingUpgrades() method:**
- Select invitations WHERE payment_status = 'pending' AND deleted_at IS NULL
- Order by updated_at ascending (oldest first)
- Also join/include user_id so admin can see who requested
- Return list of mapped invitations
