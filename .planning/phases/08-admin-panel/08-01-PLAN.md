Create a new `AdminModule` at `apps/api/src/admin/`:

**admin.module.ts**: Standard NestJS module importing SupabaseModule, exporting AdminService. Register in AppModule imports.

**admin.controller.ts**: All routes under `@Controller('admin')`, all guarded with `@UseGuards(JwtGuard, AdminGuard)` at the class level. Endpoints:

1. `GET /admin/stats` -- returns AdminStats (total users, invitations, premium count, 30-day chart data as daily counts)
2. `GET /admin/users?search=&status=&page=&limit=` -- paginated user list with invitation counts. Search by email. Filter by status (all/active/locked). Default limit=20.
3. `GET /admin/users/:id` -- single user detail with their invitations list
4. `POST /admin/users/:id/lock` -- set is_locked=true
5. `POST /admin/users/:id/unlock` -- set is_locked=false
6. `DELETE /admin/users/:id` -- hard delete: delete all user's invitation photos from `invitation-photos` bucket, bank QR from `bank-qr` bucket, QR codes from `qr-codes` bucket, then delete all invitations (cascade), then delete user. Return `{ deleted: true }`.
7. `POST /admin/users/:id/role` -- body `{ role: 'user' | 'admin' }`. Prevent admin from changing their own role (compare JWT sub with param id).
8. `GET /admin/invitations?search=&status=&plan=&page=&limit=` -- paginated invitation list with user email joined. Search by groom_name, bride_name, or slug. Filter by status and plan.
9. `GET /admin/invitations/:id` -- single invitation detail (full data, read-only)
10. `POST /admin/invitations/:id/disable` -- set is_disabled=true, triggers ISR revalidation if slug exists
11. `POST /admin/invitations/:id/enable` -- set is_disabled=false, triggers ISR revalidation if slug exists
12. `GET /admin/music-tracks` -- all tracks (including inactive), with usage_count
13. `POST /admin/music-tracks` -- upload MP3 file via `@UseInterceptors(FileInterceptor('file'))`. Validate magic bytes for audio/mpeg. Upload to `system-music` bucket. Create row in system_music_tracks with title from body, artist from body, url from storage public URL. Use multer 10MB max size.
14. `POST /admin/music-tracks/:id/toggle` -- flip is_active boolean
15. `DELETE /admin/music-tracks/:id` -- only if usage_count = 0, else throw BadRequest. Delete from storage and DB.
16. `GET /admin/themes` -- return hardcoded array of 3 theme objects (traditional, modern, minimalist) with metadata. For v1, themes are code-defined. Read `is_active` status from system_settings key `theme_config` (default all active).
17. `POST /admin/themes/:id/toggle` -- toggle theme active status in system_settings `theme_config` key
18. `GET /admin/system-settings` -- read all system_settings rows, merge into SystemSettings shape
19. `PUT /admin/system-settings` -- update system_settings rows from body. Validate with UpdateSystemSettingsDto. Support bank QR image upload separately via `POST /admin/system-settings/bank-qr` with FileInterceptor.
20. `POST /admin/system-settings/bank-qr` -- upload admin bank QR image to `system-settings` bucket, update payment_config.bankQrUrl

**admin.service.ts**: Inject SupabaseAdminService and ConfigService. Implement all business logic. For stats, run parallel queries using Promise.all:
- `select('id', { count: 'exact', head: true })` on users (not deleted)
- `select('id', { count: 'exact', head: true })` on invitations (not deleted)
- `select('id', { count: 'exact', head: true })` on invitations where status='published'
- `select('id', { count: 'exact', head: true })` on invitations where plan='premium'
- For chart data: query invitations created in last 30 days, group by date. Use raw query or fetch all and group in JS.

For user hard delete, implement as a transaction-like sequence:
1. Fetch all user's invitations to get their IDs
2. For each invitation, list and delete files from `invitation-photos/{invId}/`, `bank-qr/{invId}/`, `qr-codes/{invId}/`
3. Delete all invitations for the user (hard delete from DB, cascade handles FK)
4. Delete the user from DB (hard delete)

For music track usage_count: query `invitations` table counting rows where `music_track_id = trackId` and `deleted_at IS NULL`.

For ISR revalidation on disable/enable: reuse the same pattern from InvitationsService -- fetch NEXT_PUBLIC_URL and REVALIDATION_SECRET from config, POST to /api/revalidate. Extract into a shared utility or duplicate the private method.

**dto/update-system-settings.dto.ts**: Use class-validator. Validate nested objects with @ValidateNested and @Type. All fields optional (partial update).

**app.module.ts**: Add `AdminModule` to imports array.

Important patterns to follow:
- Use `SupabaseAdminService` (service role) for all queries -- admin bypasses RLS
- Use `ParseUUIDPipe` on all `:id` params
- Return Vietnamese error messages
- Use `@HttpCode(HttpStatus.OK)` on POST endpoints that are not creating resources
