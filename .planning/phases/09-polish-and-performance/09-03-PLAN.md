    1. Create `DesktopFrame.tsx`:
       - Accepts `templateId: TemplateId` and `children: React.ReactNode`
       - Wraps children in a centered container: `min-h-screen flex items-start justify-center py-0 md:py-12`
       - Inner card: `relative w-full max-w-full md:max-w-[480px]`
       - Template-specific FRAME_COLORS record mapping templateId to frame palette:
         - traditional: gold (#d4a843) corners, burgundy (#8B0000) vine accents, warm cream (#FFF8F0) background
         - modern: rose gold (#B76E79) corners, pink (#FFB7C5) vine accents, blush (#FFF5F7) background
         - minimalist: gray (#A0A0A0) corners, muted (#D5D0C4) vine accents, off-white (#FAFAF8) background
       - Desktop background: `hidden md:block` applied to a background div behind the card, using `bg` color from FRAME_COLORS
       - Four decorative corner elements (top-left, top-right, bottom-left, bottom-right): positioned absolute with `hidden md:block`. Use inline SVG paths for simple botanical line art (leaves/vine curves). Each corner is ~32x32px, rotated appropriately for each position. SVG uses frame corner color. Keep total SVG payload < 2KB.
       - Subtle vine border between corners: thin (1-2px) gradient line using vine color, `hidden md:block`
       - On mobile (below md): no decorative elements render, children get full width
       - The frame decorations should feel like watercolor florals -- use soft gradients and opacity in SVG fills

    2. Update `InvitationShell.tsx`:
       - Import DesktopFrame (static import, not dynamic -- it's just CSS, no heavy JS)
       - Wrap ALL content (both envelope stage and revealed stage) inside `<DesktopFrame templateId={invitation.templateId}>`
       - Change FallingPetals `enabled` prop from `{true}` to `{!envelopeOpened}` -- petals play during envelope stage only, then stop after reveal per CONTEXT.md decision
       - Add `text-base` class to the revealed stage content wrapper to ensure 16px base font size on public page only (scoped to InvitationShell, does NOT affect editor or admin). This goes on the `<div className="relative min-h-screen">` container. Note: Tailwind v4 `text-base` = 1rem = 16px.
       - Move FallingPetals inside the envelope stage conditional block (before envelopeOpened) so it only renders during envelope, not after reveal. This is cleaner than using the enabled prop -- component fully unmounts.

    3. The envelope animation on desktop should feel phone-sized: the existing EnvelopeAnimation is fixed/centered and 280x200px which is already phone-sized. The DesktopFrame just provides the visual context around it. The decorative frame should be visible BEHIND the envelope during the opening stage (z-index: frame < envelope overlay).
