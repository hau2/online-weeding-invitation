  <action>
    **Frontend: AvatarUpload.tsx**

    Create `AvatarUpload.tsx` as a `'use client'` component following the BankQrUpload pattern for uploads.

    **Props interface:**
    ```typescript
    interface AvatarUploadProps {
      invitationId: string
      avatarUrl: string | null
      label: string  // "Chu re" or "Co dau"
      endpoint: 'groom-avatar' | 'bride-avatar'
      onUploaded: (url: string) => void
    }
    ```

    **Implementation:**
    - Circular container: `w-20 h-20 rounded-full overflow-hidden border-2 border-gray-200`
    - When no avatar: gray-100 background with Camera icon (lucide-react) centered
    - When avatar exists: `<img>` tag with `object-cover` filling the circle (use regular img, not next/image, to avoid domain config issues for Supabase URLs — same pattern as templates)
    - Hover overlay: semi-transparent black with Camera icon
    - Hidden file input, triggered by clicking the circle
    - On file select: call `apiUpload<Invitation>(`/invitations/${invitationId}/${endpoint}`, file)` — same pattern as BankQrUpload
    - Show loading spinner during upload (use a simple `animate-spin` on the camera icon)
    - On success: call `onUploaded(data.groomAvatarUrl || data.brideAvatarUrl)` to update parent state
    - On error: `toast.error(error)` via sonner
    - Below the circle: small gray text label (e.g., "Chu re")
    - Accept image files only: `accept="image/jpeg,image/png,image/webp"`

    **Backend: Avatar upload endpoints**

    Add to `invitations.controller.ts`:
    - `POST :id/groom-avatar` — follows exact pattern of existing `POST :id/bank-qr` endpoint
    - `POST :id/bride-avatar` — same pattern
    - Both use `@UseInterceptors(FileInterceptor('file'))` with same multer config as bank-qr
    - Both call corresponding service methods

    Add to `invitations.service.ts`:
    - `async uploadGroomAvatar(userId: string, id: string, file: Express.Multer.File): Promise<Invitation>` — follows uploadBankQr pattern exactly:
      1. Validate ownership (findOneForUser)
      2. Validate MIME type via magic-bytes
      3. Process image with `sharp.resize(400, 400, { fit: 'cover' }).webp()` (square crop for avatar)
      4. Upload to Supabase storage bucket 'invitation-photos' at path `${id}/groom-avatar.webp` (reuse existing bucket, avatar sub-path)
      5. If previous avatar exists, delete old file first (upsert pattern from bank-qr)
      6. Update `groom_avatar_url` column
      7. Return full invitation via SELECT_ALL + mapRow
    - `async uploadBrideAvatar(...)` — same logic with `bride_avatar_url` and path `${id}/bride-avatar.webp`

    Note: Use existing 'invitation-photos' bucket rather than creating a new 'avatars' bucket (simpler, follows existing patterns, no new bucket policy needed).
